v0.411 (2026-03-24)
───────────────────
FIX:
  - ROOT CAUSE FOUND: Windows synchronous pipe handles serialize ALL I/O.
    ReadFile blocks → WriteFile from another thread ALSO BLOCKS → deadlock.
    The background reader thread was the problem.
  - Removed reader thread entirely. All pipe I/O is now synchronous
    request/response: write request, read response, sequentially.
    Matches how pipe-test.exe works (which always worked).
  - Thread-safe via single pipe lock for all I/O operations.

v0.408 (2026-03-23)
───────────────────

  - WriteFile buffer: Python bytes can't pass to c_void_p — use

  - Appearance mode buttons: null-guard broke ternary operator,
    all three buttons rendered as black. Fixed comparison logic.

v0.407 (2026-03-23)
───────────────────
FIX:
  - Named pipe I/O rewritten to use ctypes (CreateFileW/ReadFile/WriteFile).
    Python's open() uses the C runtime which doesn't support concurrent
    read+write on Windows named pipes — caused get_status to always time out,
    "Not Connected" status, and DPI commands never reaching the service.
  - Reader thread now uses 4KB buffered ReadFile instead of byte-by-byte read.

WHAT MASTERMICE ADDS OVER UPSTREAM MOUSER (github.com/TomBadash/Mouser):
  Upstream Mouser focuses on multi-device breadth (8 mice, data-driven layouts,
  app catalog). MasterMice focuses on deep HID++ hardware control for MX Master
  series mice. The codebases diverged significantly — these features do NOT exist
  in upstream Mouser:

  HID++ Hardware Control:
    - SmartShift v1 (MX3) + v2 (MX4): threshold, scrolling force, on/off toggle
    - Hi-Res Scroll toggle with multiplier detection
    - Haptic Feedback control (MX4): on/off + intensity slider
    - Smooth Scrolling toggle (device-side, per-device ON values)
    - Windows Pointer Speed (SPI_SETMOUSESPEED) + Scroll Lines control
    - Thread-safe _queued_request() pattern for all HID++ I/O

  MX Master 4 Support:
    - Actions Ring button (CID 0x01A0) divert and capture
    - Haptic Sense Panel control (0xB019)
    - UNIFIED_BATTERY (0x1004) with ext_power charging detection
    - SmartShift Enhanced (0x2111) with force parameter
    - Auto-detection via feature fingerprinting (haptics→MX4, unified→MX3S, level→MX3)

  Battery System:
    - Dual battery feature support (0x1000 for MX3, 0x1004 for MX4)
    - MX3 charging quirk: caches last known % (device reports 0% while charging)
    - MX4 status 8 = "normal" (not an error)
    - Battery icon with charging animation, color-coded by level

  Connection & Compatibility:
    - Auto-detection of mouse model via DEVICE_NAME + feature fingerprinting
    - Connection type indicator (Unifying/Bolt/Bluetooth) from receiver PID
    - Logitech software conflict detection on startup (Options+/SetPoint warning)
    - Skip 0xFFBC interface (receiver management, never has device HID++)
    - fix_receiver.ps1 tool for Code 28 driver issues

  Diagnostics & Logging:
    - Rotating log file with configurable level (disabled/errors/verbose) + max size
    - Built-in HID++ diagnostics with progress bar
    - Standalone debug tools: battery_test, mouse_diag_v2, smartshift_test, hid_debug
