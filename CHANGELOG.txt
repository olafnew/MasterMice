──────────────────

  - Go Windows service (mastermice-svc.exe) for HID++ device communication

  - Service auto-kills Logitech Options+ before opening HID++ handles

  - mouse_hook.py: HidGestureListener creation skipped (Go service owns HID++)
  - New file: core/service_client.py — named pipe client for Go service
  - pipe-test.exe — standalone IPC test client
NEW:

    Hi-Res wheel, smooth scroll, haptic motor (MX4)
  - Interactive console in service: dpi/smartshift/hires/smooth commands
  - Device auto-detection via DEVICE_NAME + profile matching
  - get_status, get_capabilities, health commands via IPC
  - JSON-lines protocol with request/response ID matching and event broadcasting
VALIDATED:
  - MX Master 3 on Unifying receiver: all features read + write confirmed
  - Named pipe IPC: health, status, battery, dpi, smartshift via pipe-test.exe

v0.35 (2026-03-20)
───────────────────
NEW:
  - Haptic vibration motor support (MX Master 4 ONLY)!
    * Feature 0x19B0 at index 0x0B — uses HidD_SetOutputReport on SHORT
      HID collection (Usage=0x0001) via ctypes, NOT hidapi.write().
    * Bolt receivers ignore interrupt OUT for HID++ — must use USB SET_REPORT
      control transfers. Discovered via full USB capture analysis.
    * haptic_set_config(enabled, intensity) — enable/disable + intensity 0-100%
    * haptic_trigger(pulse_type) — fire motor: 0x02=light, 0x04=tick, 0x08=strong
    * Test button in Haptic Feedback card sends strong pulse
    * Intensity presets: Subtle(25%), Low(45%), Medium(60%), High(100%)
  - Button press sensitivity control (feature 0x19C0 at index 0x0C)
    * Light/Medium/Hard/Firm presets via LONG report SetOutputReport
  - Dual HID handle architecture: SHORT handle (haptic) + LONG handle (HID++ 2.0)
    * hidapi for device finding (PyInstaller compatible)
    * ctypes CreateFileW + HidD_SetOutputReport for sending (correct USB pipe)
FIX:
  - Haptic feature discovery no longer depends on IRoot probe for 0xB019 or
    FEATURE_SET enumeration — handled via profile-driven SHORT handle opening

WHAT MASTERMICE ADDS OVER UPSTREAM MOUSER (github.com/TomBadash/Mouser):
  Upstream Mouser focuses on multi-device breadth (8 mice, data-driven layouts,
  app catalog). MasterMice focuses on deep HID++ hardware control for MX Master
  series mice. The codebases diverged significantly — these features do NOT exist
  in upstream Mouser:

  HID++ Hardware Control:
    - SmartShift v1 (MX3) + v2 (MX4): threshold, scrolling force, on/off toggle
    - Hi-Res Scroll toggle with multiplier detection
    - Haptic Feedback control (MX4): on/off + intensity slider
    - Smooth Scrolling toggle (device-side, per-device ON values)
    - Windows Pointer Speed (SPI_SETMOUSESPEED) + Scroll Lines control
    - Thread-safe _queued_request() pattern for all HID++ I/O

  MX Master 4 Support:
    - Actions Ring button (CID 0x01A0) divert and capture
    - Haptic Sense Panel control (0xB019)
    - UNIFIED_BATTERY (0x1004) with ext_power charging detection
    - SmartShift Enhanced (0x2111) with force parameter
    - Auto-detection via feature fingerprinting (haptics→MX4, unified→MX3S, level→MX3)

  Battery System:
    - Dual battery feature support (0x1000 for MX3, 0x1004 for MX4)
    - MX3 charging quirk: caches last known % (device reports 0% while charging)
    - MX4 status 8 = "normal" (not an error)
    - Battery icon with charging animation, color-coded by level

  Connection & Compatibility:
    - Auto-detection of mouse model via DEVICE_NAME + feature fingerprinting
    - Connection type indicator (Unifying/Bolt/Bluetooth) from receiver PID
    - Logitech software conflict detection on startup (Options+/SetPoint warning)
    - Skip 0xFFBC interface (receiver management, never has device HID++)
    - fix_receiver.ps1 tool for Code 28 driver issues

  Diagnostics & Logging:
    - Rotating log file with configurable level (disabled/errors/verbose) + max size
    - Built-in HID++ diagnostics with progress bar
    - Standalone debug tools: battery_test, mouse_diag_v2, smartshift_test, hid_debug
    - Every setting change logged, every error logged

  UI:
    - 10 callout dots (MX3) / 11 callout dots (MX4) with non-crossing line algorithm
    - Dark mode with automatic line-art inversion for mouse images
    - Settings page with logging, diagnostics, device info, light/dark mode
    - Device selector on first run (replaced by auto-detection)
    - DPI presets filtered by device max (4000 for MX3, 8000 for MX4)

v0.28 (2026-03-18)
───────────────────
FIX:
  - SmartShift v1 (MX3) NOW WORKS — correct param format is [auto_disengage, threshold],
    not [threshold] alone. Confirmed via standalone debug tool with Wireshark-verified
    protocol. auto_disengage is always 10 on MX3.
  - Thread safety: all HID++ getters/setters (SmartShift, HiRes, Haptic, Smooth Scroll)
    now use a queued command pattern — requests execute on the listener thread only.
    Previously, UI thread called _request() directly while the listener thread was also
    reading from the same HID device, causing response collisions and intermittent failures.
  - HiRes wheel SET no longer fails intermittently (same threading fix).
NEW:
  - tools/smartshift_test.py — standalone interactive SmartShift & HiRes debug tool
    with raw hex logging for protocol debugging.

v0.27 (2026-03-18)
───────────────────

  - SmartShift v1 (MX3) SET was broken — used func=0 (read-only) instead of func=1.

    threshold-only param matching original working code.

    8000 on MX3 which returned HID++ error 0x02 (invalid argument).
  - Hi-Res scrolling toggle now explains: "Sends 15x more scroll events per notch —
